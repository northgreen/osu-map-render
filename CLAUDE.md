## Project Overview

This is a Remotion project for rendering osu!mania beatmap videos. Remotion is a framework for creating videos programmatically using React.

## Commands

```bash
npm run dev                          # Start Remotion Studio (live preview)
npm run build                        # Bundle the project (auto-runs parse)
npm run lint                         # Run ESLint + TypeScript
npm run parse                        # Parse beatmap to JSON (run manually if needed)
npm run parse:replay                 # Parse .osr replay file to JSON
npx remotion render ManiaRender out/video.mp4  # Render full osu!mania video
npx remotion render ManiaStageOnly out/video.mp4  # Stage layer only (notes, hit effects)
npx remotion render ManiaBackground out/video.mp4  # Background layer only
npx remotion render ManiaOverlayOnly out/video.mp4  # Overlay only (score, metadata)
npm run upgrade                      # Upgrade Remotion version
```

## Architecture

### Layer Structure
The video renders in layers from back to front:
1. **Background** (`ManiaBackground.tsx`) - Static background image/color
2. **Stage** (`ManiaStageLayer.tsx`) - Notes, judgment line, key press effects, hit effects
3. **Overlay** (`ManiaOverlay.tsx`) - Score, combo, accuracy, metadata display
4. **Replay Cursor** (`ReplayCursorLayer.tsx`) - Falling key press indicators from replay data

### Key Files
- **Entry point**: `src/index.ts` - registers the root component
- **Compositions**: `src/Root.tsx` - defines video compositions using `<Composition>`
- **Beatmap data**: `src/lib/beatmap.json` - parsed beatmap data (generated by parse script)
- **Replay data**: `src/lib/replay.json` - parsed replay data (generated by parse:replay)
- **Parser script**: `scripts/parseBeatmap.ts` - parses .osu file to JSON
- **Mania renderer**: `src/ManiaRender.tsx` - combined render of all layers

### Available Compositions
- `ManiaRender` - Full render (background + stage + overlay)
- `ManiaBackground` - Background only
- `ManiaStageOnly` - Stage layer only (notes, judgment line, key presses, hit effects)
- `ManiaOverlayOnly` - Overlay only (score, combo, accuracy, metadata)
- `ManiaReplayCursorOnly` - Replay cursor layer only

## Configuration

All customization options are in `src/config.ts` and as props to ManiaRender components:

### Config.ts Values
- `SCROLL_SPEED` - Note fall speed (default: 37)
- `BASE_VISIBLE_TIME` - Base visible time at scroll speed 10 (ms, default: 1800)
- `NOTE_WIDTH`, `NOTE_HEIGHT` - Note dimensions (default: 100x40)
- `COLUMN_POSITIONS_STAGE`, `COLUMN_POSITIONS_NOTE` - Column X positions
- `JUDGMENT_LINE_Y` - Judgment line Y position (default: 900)
- `COLUMN_COLORS` - Colors for each column
- `HIT_EFFECT_DURATION` - Hit effect fade duration (ms, default: 300)
- `LN_BODY_OPACITY` - Long note body opacity (default: 0.4)

### Render Props (passed to compositions)
- `scrollSpeed` - Note scroll speed (5-50, default: 20)
- `timeOffset` - Time offset in ms
- `beatOffset` - Beat offset from first timing point (default: 900ms)
- `judgmentMode` - "v1" (Classic) or "v2" (ScoreV2, default)
- `judgmentOffset` - Judgment timing offset
- `showJudgmentZones` - Show judgment zone indicators
- `stageOffset` - Additional stage offset
- `judgmentLineY` - Judgment line Y position (100-1000, default: 900)

## Judgment System

Located in `src/lib/judgment.ts`:
- Supports two modes: **v1** (Classic) and **v2** (ScoreV2)
- Hit windows calculated based on Overall Difficulty (OD)
- Judgment grades: Perfect (320), Great (300), Good (200), Ok (100), Meh (50), Miss
- LN tail judgment supported
- Replay-based automatic judgment calculation from key press events

## Replay Data (.osr)

To include player replay data in renders:
1. Place your .osr file in `cheart/` folder
2. Update the filename in `scripts/parseReplay.ts`
3. Run `npm run parse:replay` to parse to JSON
4. Replay data is auto-imported in `src/lib/replay.ts`

Use `src/lib/osrParser.ts` to parse replay files:
- `parseOsuReplay(path)` - Parse .osr file
- `ReplayFrame` - Contains timeOffset, x, y, keys (bitmask)
- Key constants: `KEY_M1`, `KEY_M2`, `KEY_K1`, `KEY_K2`, `KEY_SMOKE`
